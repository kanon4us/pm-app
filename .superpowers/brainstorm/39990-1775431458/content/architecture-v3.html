System Architecture — v3

ClickUp → PM Agent → four write-back targets

── CLICKUP ─────────────────────────────────────────────────────
OAuth 2.0 → user signs in with their ClickUp account
REST API → import tasks from up to 10 selected lists
Webhooks → ClickUp POSTs status-change events to our endpoint
▼ webhook POST / REST calls
── NEXT.JS APP (Vercel) ────────────────────────────────────────
API Routes
POST /api/webhooks/clickup → receives status-change events
POST /api/triggers/approve → developer approves a queued trigger
POST /api/triggers/dismiss → developer dismisses a trigger
GET /api/lists → fetch available ClickUp lists
Pages
/ → Trigger Queue dashboard (main view)
/setup → ClickUp auth + list selection
/triggers/config → Configure which status changes fire which PM Agent action
▼ reads/writes / realtime subscription
── SUPABASE (Postgres + Realtime) ──────────────────────────────
tables: tasks, trigger_queue, trigger_configs, agent_runs
Realtime channel → dashboard sees new triggers instantly
▼ on developer approval
── CLAUDE API (claude-opus-4-6) ────────────────────────────────
System prompt = PM Agent SKILL.md + task context + ROI evaluation mode
Structured output: { clickup, docs_branch, webflow, figma } write-back payloads
▼ result fans out to four targets
── WRITE-BACK TARGETS ──────────────────────────────────────────
① ClickUp
Comment (always) + description / custom fields update (slug, risk, ROI score)
② Documentation Repo (GitHub)
Create branch feature/[slug] — spec, CLAUDE.md, user stories
Stays unmerged until ROI is validated
③ Webflow CMS
Draft post or Coming Soon stub (published: false until shipped)
④ Figma
Create new page or file for the feature's UI/UX design work
Linked back to the ClickUp task and docs branch
