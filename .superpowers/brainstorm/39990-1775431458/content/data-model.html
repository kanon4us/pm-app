RE: Priority question

trigger_configs.write_back_order (text[]) defines execution sequence — e.g. ["clickup", "docs", "webflow", "figma"].

trigger_configs.on_failure (enum: continue|stop) controls behavior when a target fails:
  continue — log error to trigger_queue.error_details, keep writing to remaining targets
  stop — halt the run, surface in dashboard for developer review

Default: continue — ClickUp comment always lands even if Figma API limit is hit. Orange fields = additions from your review. Green = original schema.
