④ Figma — bidirectional via MCP (see phase map below)
FIGMA × WORKFLOW PHASE MAP
Phase Figma Action Direction
UI/UX Design PM Agent generates coded mockup → push to Figma as starting point Code → Canvas
UI/UX Review Send current build to Figma review file, highlight discrepancies Code → Canvas
Architecting Designer updates component → PM Agent pulls spec → updates code Canvas → Code
QA Approved Push Master Design to Figma — visual docs match what shipped Code → Canvas
⚠ SCOPE FLAG

"Code → Canvas" requires running the app locally (localhost) so the agent can preview the UI before pushing frames to Figma. This only works when the developer triggers it from their local machine — it cannot run automatically from Vercel's server. This is a Phase 2 capability. Phase 1 ships Canvas → Code and the Figma page/frame creation without the localhost requirement.
